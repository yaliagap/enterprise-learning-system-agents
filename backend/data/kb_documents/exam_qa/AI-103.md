## Skill Area: Plan and manage Azure AI solutions (15–20%)

### Q1
**Question:** An AI engineer needs to provision Azure AI services for a new project. The team requires a single endpoint and API key to access multiple AI capabilities (Vision, Language, Speech, Document Intelligence). Which resource type should be deployed?

**Options:**
- A. Individual resource per service (Cognitive Services Vision, Language, Speech separately)
- B. Azure AI Services multi-service account (formerly Cognitive Services)
- C. Azure OpenAI Service resource
- D. Azure AI Studio hub resource

**Correct Answer:** B  
**Explanation:** The Azure AI Services multi-service account provides a single endpoint, API key, and billing entity for accessing Vision, Language, Speech, Document Intelligence, Content Safety, and other Azure AI capabilities. This simplifies management for applications that use multiple AI services. Azure OpenAI Service is a separate resource for GPT/DALL-E/Whisper models only.

### Q2
**Question:** An enterprise team is building multiple AI solutions that share common infrastructure: Azure OpenAI deployments, vector indexes in Azure AI Search, and content safety configurations. Which Azure AI Foundry resource should be the organizational unit for shared infrastructure?

**Options:**
- A. Azure AI Foundry Project (for each team's work)
- B. Azure AI Foundry Hub (shared infrastructure across projects)
- C. Azure AI Services resource group
- D. Azure OpenAI Service resource

**Correct Answer:** B  
**Explanation:** Azure AI Foundry uses a Hub-and-Project model. The Hub is the organizational unit for shared infrastructure: connected resources (Azure OpenAI, AI Search, Storage, Key Vault), network and security configurations, and compute. Projects inherit hub resources and are the working environment for individual teams or AI applications. Multiple projects share one hub's infrastructure.

## Skill Area: Implement generative AI and agentic solutions (25–30%)

### Q3
**Question:** An AI engineer is building a RAG (Retrieval-Augmented Generation) chatbot that must answer questions based on a company's internal knowledge base of 50,000 documents. Which Azure AI Foundry component is used to index these documents for semantic search?

**Options:**
- A. Azure Blob Storage with keyword search
- B. Azure AI Search with vector indexes and semantic ranking
- C. Azure Cosmos DB for NoSQL with full-text search
- D. Azure SQL Database with CONTAINS full-text search

**Correct Answer:** B  
**Explanation:** Azure AI Search supports vector embeddings (using embedding models from Azure OpenAI) stored in vector fields, enabling semantic similarity search across documents. Combined with hybrid search (keyword + vector) and semantic ranking (L2 reranker), it provides state-of-the-art retrieval quality for RAG applications. Documents are chunked, embedded, and indexed; queries are embedded at runtime and matched by cosine similarity.

### Q4
**Question:** An AI engineer needs to build an agent that can autonomously execute multi-step tasks: searching the web, reading files, calling APIs, and writing results to a database — with memory across turns. Which Azure AI Foundry service provides the infrastructure for this agentic workflow?

**Options:**
- A. Azure OpenAI Service with function calling only
- B. Azure AI Agent Service (Azure AI Foundry Agents)
- C. Azure Logic Apps with AI connectors
- D. Azure OpenAI Assistants API with Retrieval tool

**Correct Answer:** B  
**Explanation:** Azure AI Agent Service (part of Azure AI Foundry) provides the infrastructure for building production-grade AI agents: thread management for multi-turn memory, tool orchestration (code interpreter, file search, function calling, Bing grounding), run management, and built-in observability. It abstracts the agent loop, allowing engineers to focus on tools and prompts rather than orchestration plumbing.

### Q5
**Question:** When building an AI agent using the Semantic Kernel SDK, what is a "plugin"?

**Options:**
- A. A VS Code extension for debugging AI agents
- B. A collection of functions (native or prompt-based) that an AI agent can invoke to extend its capabilities
- C. A Kubernetes container that hosts the AI model
- D. An authentication module for connecting to Azure AI services

**Correct Answer:** B  
**Explanation:** In Semantic Kernel, a Plugin is a grouping of functions that an agent (or the kernel) can discover and invoke. Plugins can be native functions (C#/Python code) or prompt functions (LLM calls with a templated prompt). The AI model selects which plugin functions to call based on the user's request — analogous to tools in other agent frameworks. This is the core extensibility mechanism for Semantic Kernel agents.

### Q6
**Question:** An AI engineer notices that a GPT-4o model sometimes generates answers that directly contradict the retrieved documents in a RAG pipeline. What is this behavior called, and which Azure AI Content Safety feature helps detect it?

**Options:**
- A. Prompt injection; Prompt Shields
- B. Hallucination; Azure AI Content Safety Groundedness Detection
- C. Data drift; Azure ML model monitoring
- D. Model bias; Azure AI Fairness assessment

**Correct Answer:** B  
**Explanation:** Hallucination occurs when an LLM generates content that contradicts or is not supported by the provided source documents. Azure AI Content Safety's Groundedness Detection analyzes model outputs against retrieved source documents and flags claims not grounded in the sources. This is critical for RAG applications where answer accuracy and source attribution are required.

## Skill Area: Implement computer vision solutions (10–15%)

### Q7
**Question:** An AI engineer needs to extract structured data from scanned invoices — vendor name, invoice date, line items, and total amounts — from thousands of PDFs daily. The invoices vary in layout across different vendors. Which Azure AI service and model handles this?

**Options:**
- A. Azure AI Vision OCR (Read API) with regex post-processing
- B. Azure AI Document Intelligence — Invoice prebuilt model
- C. Azure OpenAI GPT-4o with Vision for each invoice
- D. Azure Custom Vision with document classification

**Correct Answer:** B  
**Explanation:** Azure AI Document Intelligence's Invoice prebuilt model is specifically trained to extract structured fields from invoices: vendor name, customer info, invoice ID, date, line items, tax, and total. It handles diverse layouts using the Document Analysis engine. This is far more accurate and cost-effective than OCR + regex or GPT-4o Vision for high-volume invoice processing.

## Skill Area: Implement natural language processing solutions (20–25%)

### Q8
**Question:** An AI engineer is building a customer support chatbot that must handle 15 languages and route conversations based on the customer's intent (billing inquiry, technical support, returns). The solution should be managed centrally in Azure AI Language Studio. Which feature combination addresses this?

**Options:**
- A. Custom Text Classification + Language Detection
- B. Conversational Language Understanding (CLU) with multilingual project + Language Detection
- C. Azure OpenAI function calling + translation
- D. QnA Maker + bot framework

**Correct Answer:** B  
**Explanation:** Conversational Language Understanding (CLU) supports multilingual projects — a single model trained on utterances in one language can generalize to 15+ languages. Language Detection identifies the user's language, but CLU projects handle multilingual intent extraction natively. QnA Maker is deprecated (succeeded by Question Answering in Azure AI Language).

### Q9
**Question:** A media company needs to process 10,000 customer feedback emails daily and summarize each one to two sentences for a dashboard. The summaries must retain the key sentiment and main topic. Which Azure AI Language capability should be used?

**Options:**
- A. Key Phrase Extraction
- B. Extractive Summarization (selects sentences from original text) or Abstractive Summarization (generates new summary sentences)
- C. Sentiment Analysis with opinion mining
- D. Named Entity Recognition

**Correct Answer:** B  
**Explanation:** Azure AI Language provides two summarization approaches: Extractive Summarization selects the most representative sentences from the original text; Abstractive Summarization generates new sentences that capture the key information. For a two-sentence daily dashboard summary, abstractive summarization produces more natural, concise output. Both are available as batch processing APIs suitable for 10,000 documents/day.

## Skill Area: Implement knowledge mining and document intelligence (10–15%)

### Q10
**Question:** An organization has 2 million scanned legal documents in Azure Blob Storage. They want to enable full-text search, automatically extract entities (organizations, dates, case numbers), and enable faceted filtering by document type and year. Which Azure service architecture is recommended?

**Options:**
- A. Azure AI Search with a blob indexer and built-in AI enrichment skillset (OCR + entity extraction)
- B. Azure SQL Database full-text search with manual data loading
- C. Azure Cosmos DB with Azure Synapse Analytics for search queries
- D. Azure OpenAI with a vector database for semantic search only

**Correct Answer:** A  
**Explanation:** Azure AI Search's indexer pipeline for blob storage: (1) Blob indexer crawls the storage container; (2) AI enrichment skillset applies OCR (for scanned PDFs), entity recognition, key phrase extraction, and custom skills; (3) Enriched content is indexed in a search index with full-text fields, entity fields, and facetable fields for filtering. This is the reference architecture for knowledge mining on unstructured document collections.

## Skill Area: Apply responsible AI and content safety (10–15%)

### Q11
**Question:** An AI engineer is deploying a customer-facing chatbot powered by Azure OpenAI. The solution must prevent the chatbot from generating harmful content categories (violence, hate speech, sexual content) and block user attempts to "jailbreak" the system prompt. Which Azure AI service provides both capabilities?

**Options:**
- A. Azure OpenAI system prompt instructions alone
- B. Azure AI Content Safety — Content Filters + Prompt Shields
- C. Microsoft Defender for Cloud AI threat protection
- D. Azure Policy with AI compliance initiative

**Correct Answer:** B  
**Explanation:** Azure AI Content Safety provides: (1) Content filters — configured severity thresholds for harmful categories (violence, hate, sexual, self-harm) that automatically block or flag content in both user messages and model responses; (2) Prompt Shields — detect and block direct jailbreak attempts (user prompt attacks) and indirect prompt injection (content in retrieved documents or tool outputs that attempts to override system instructions). These work as pre/post-processing layers around the model call.
