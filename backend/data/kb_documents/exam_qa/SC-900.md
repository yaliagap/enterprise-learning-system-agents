## Skill Area: Describe the concepts of security, compliance, and identity (10–15%)

### Q1
**Question:** The Zero Trust security model operates on which core principle?

**Options:**
- A. Trust all users inside the corporate network perimeter
- B. Verify explicitly, use least privilege access, and assume breach
- C. Block all external access and allow only VPN-authenticated sessions
- D. Trust devices that are domain-joined to the corporate Active Directory

**Correct Answer:** B  
**Explanation:** Zero Trust is built on three principles: (1) Verify explicitly — always authenticate and authorize based on all available signals (identity, location, device, service, workload, data classification); (2) Use least privilege access — limit user access with just-in-time and just-enough-access, adaptive policies, and data protection; (3) Assume breach — minimize blast radius, segment access, encrypt everything, use analytics to detect threats.

### Q2
**Question:** What does the CIA triad stand for in information security?

**Options:**
- A. Cryptography, Identity, Authentication
- B. Confidentiality, Integrity, Availability
- C. Compliance, Investigation, Authorization
- D. Certificates, Infrastructure, Access

**Correct Answer:** B  
**Explanation:** The CIA triad is the foundational security model: Confidentiality ensures information is accessible only to authorized parties; Integrity ensures data is accurate and has not been tampered with; Availability ensures systems and data are accessible to authorized users when needed. All security controls are designed to protect one or more of these properties.

---

### Q3
**Question:** Which of the following best describes "defense in depth"?

**Options:**
- A. A single strong firewall that blocks all external attacks
- B. A layered security approach where multiple controls protect data even if one layer is breached
- C. Encrypting all data at rest and in transit
- D. Using multi-factor authentication as the only security control

**Correct Answer:** B  
**Explanation:** Defense in depth applies multiple overlapping security layers: physical security, identity/access, perimeter (DDoS, firewall), network (NSGs, segmentation), compute (endpoint protection, patching), application (WAF, SAST), and data (encryption, classification). If one layer fails, others continue to protect. No single control is sufficient.

## Skill Area: Describe the capabilities of Microsoft Entra (25–30%)

### Q4
**Question:** A user tries to sign in to a corporate application from an unmanaged personal device in an unfamiliar country. Microsoft Entra ID evaluates signals and prompts for additional verification before granting access. What feature is responsible for this behavior?

**Options:**
- A. Password Hash Synchronization
- B. Conditional Access Policies
- C. Privileged Identity Management
- D. Self-Service Password Reset (SSPR)

**Correct Answer:** B  
**Explanation:** Conditional Access evaluates sign-in signals (user, device compliance, location, application, real-time risk) and enforces access controls. For high-risk sign-ins (unusual location, non-compliant device), policies can require MFA, limit access to compliant devices, or block access entirely. This is the core Conditional Access scenario tested on SC-900.

---

### Q5
**Question:** An IT administrator wants to ensure that users who are assigned the Global Administrator role in Microsoft Entra ID only activate that role when needed, and must provide a justification and receive approval before activation. Which feature enables this?

**Options:**
- A. Role-Based Access Control (RBAC)
- B. Microsoft Entra Privileged Identity Management (PIM)
- C. Conditional Access Policy with MFA
- D. Access Reviews

**Correct Answer:** B  
**Explanation:** Microsoft Entra PIM provides just-in-time privileged access. Eligible assignments allow users to activate roles for a limited time (e.g., 1–8 hours), requiring MFA and business justification. Approvals can be required from designated approvers. All activations are logged for audit purposes.

### Q6
**Question:** What is the purpose of Microsoft Entra External ID?

**Options:**
- A. Managing identities for internal employees only
- B. Providing identity and access management for external users such as customers, partners, and guests (B2B and B2C scenarios)
- C. Synchronizing on-premises AD with Microsoft Entra ID
- D. Managing device identities in Microsoft Intune

**Correct Answer:** B  
**Explanation:** Microsoft Entra External ID (formerly Azure AD B2B and Azure AD B2C) enables organizations to manage identities for people outside their organization. B2B collaboration invites partners and vendors as guests. Customer Identity and Access Management (CIAM) allows organizations to build customer-facing sign-up/sign-in experiences with social identity providers.

## Skill Area: Describe the capabilities of Microsoft security solutions (35–40%)

### Q7
**Question:** A security team needs to collect security events from Azure resources, on-premises servers, Microsoft 365, and third-party firewalls into a single platform for threat detection and automated response. Which Microsoft service fulfills this requirement?

**Options:**
- A. Microsoft Defender for Cloud
- B. Microsoft Defender for Endpoint
- C. Microsoft Sentinel
- D. Azure Monitor Log Analytics

**Correct Answer:** C  
**Explanation:** Microsoft Sentinel is a cloud-native SIEM (Security Information and Event Management) and SOAR (Security Orchestration, Automated Response) solution. It ingests logs from Azure, Microsoft 365, on-premises, and 300+ third-party connectors, detects threats using analytics rules and ML, and automates responses via Playbooks (Logic Apps).

---

### Q8
**Question:** Which Microsoft security solution provides endpoint detection and response (EDR), next-generation antivirus, threat and vulnerability management, and attack surface reduction for Windows, macOS, Linux, iOS, and Android devices?

**Options:**
- A. Microsoft Defender for Cloud
- B. Microsoft Defender for Identity
- C. Microsoft Defender for Endpoint
- D. Microsoft Defender for Office 365

**Correct Answer:** C  
**Explanation:** Microsoft Defender for Endpoint (MDE) is the enterprise endpoint security platform providing EDR, next-gen antivirus (cloud-delivered protection), attack surface reduction rules, threat and vulnerability management, and automated investigation and remediation. Defender for Identity protects on-premises AD; Defender for Cloud secures Azure resources.

---

### Q9
**Question:** What is the primary function of Microsoft Defender for Cloud in an Azure environment?

**Options:**
- A. Scanning email attachments for malware
- B. Providing a unified Cloud Security Posture Management (CSPM) and Cloud Workload Protection Platform (CWPP) for Azure, on-premises, and multi-cloud resources
- C. Encrypting data at rest in Azure Storage automatically
- D. Managing user identities and access policies

**Correct Answer:** B  
**Explanation:** Microsoft Defender for Cloud provides CSPM (identifies misconfigurations, provides Secure Score, regulatory compliance dashboards) and CWPP (workload-specific threat protection for VMs, containers, SQL, App Service, Key Vault, etc.). It works across Azure, AWS, GCP, and on-premises environments.

---

### Q10
**Question:** An organization needs protection against DDoS attacks that automatically detects and mitigates volumetric, protocol, and resource-layer attacks targeting their Azure virtual networks. Which service provides this?

**Options:**
- A. Azure Firewall Premium
- B. Azure DDoS Protection Standard (Network Protection)
- C. Azure Web Application Firewall (WAF)
- D. Network Security Groups (NSGs)

**Correct Answer:** B  
**Explanation:** Azure DDoS Protection Standard (now called Network Protection) provides always-on monitoring and automatic attack mitigation for Azure resources with public IP addresses. It includes SLA guarantees, cost protection credits, rapid response support, and telemetry in Azure Monitor. DDoS Basic (Infrastructure Protection) is free but provides only platform-level protection without customization.

## Skill Area: Describe the capabilities of Microsoft compliance solutions (20–25%)

### Q11
**Question:** A compliance officer needs to find all emails in Microsoft 365 that contain the phrase "Project Falcon" sent between January 1 and March 31, 2025, as part of a legal investigation. Which Microsoft Purview capability is used?

**Options:**
- A. Data Loss Prevention (DLP) policies
- B. Microsoft Purview eDiscovery
- C. Microsoft Purview Audit (Standard)
- D. Insider Risk Management

**Correct Answer:** B  
**Explanation:** Microsoft Purview eDiscovery allows legal and compliance teams to search, preserve, export, and review electronic data across Microsoft 365 services (Exchange, Teams, SharePoint, OneDrive) for legal hold and litigation support. Content Search (basic) and eDiscovery (Standard and Premium) provide increasingly sophisticated capabilities for legal investigations.

---

### Q12
**Question:** A company wants to automatically classify documents in SharePoint that contain credit card numbers and prevent them from being shared externally via email. Which Microsoft Purview capability handles both classification and protection?

**Options:**
- A. Microsoft Purview Sensitivity Labels + Data Loss Prevention (DLP) policies
- B. Microsoft Purview Records Management + Retention Policies
- C. Microsoft Purview Audit + eDiscovery
- D. Microsoft Purview Compliance Manager

**Correct Answer:** A  
**Explanation:** Sensitivity Labels classify and protect content with visual markings and encryption. DLP policies detect content matching conditions (like credit card number patterns) and enforce protection actions (blocking, warning, encrypting) when users attempt to share content externally via email, Teams, or cloud storage. These two features work together as the core information protection solution.
