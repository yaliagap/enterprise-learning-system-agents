## Skill Area: Manage identity, access, and governance (20–25%)

### Q1
**Question:** A security engineer needs to ensure that members of the "AzureAdmins" group can only access Azure resources during their working hours (8 AM–6 PM, Monday–Friday) and must re-authenticate if they switch to a sensitive resource. Which combination of Microsoft Entra ID features enforces this?

**Options:**
- A. Conditional Access Policy with time-based conditions + Continuous Access Evaluation (CAE)
- B. Azure RBAC time-bounded role assignments + MFA
- C. Microsoft Entra PIM with time-limited activation windows
- D. Access Reviews with monthly recertification

**Correct Answer:** A  
**Explanation:** Conditional Access supports sign-in frequency and session controls including persistent browser session settings. Time-based named locations and sign-in frequency policies enforce re-authentication windows. Continuous Access Evaluation (CAE) enables near real-time policy enforcement by having resource providers evaluate access token validity continuously rather than waiting for token expiry.

### Q2
**Question:** A company is implementing the principle of least privilege for its Azure subscription administrators. Permanent Global Admin assignments have been identified as a risk. What is the recommended remediation?

**Options:**
- A. Remove all Global Admin assignments and use service principals instead
- B. Convert permanent Global Admin assignments to eligible assignments in Microsoft Entra Privileged Identity Management (PIM) with approval workflow and MFA on activation
- C. Create separate admin accounts in a separate Azure AD tenant
- D. Enable Conditional Access to block Global Admin sign-ins from outside corporate network

**Correct Answer:** B  
**Explanation:** Microsoft Entra PIM converts standing (permanent) privileged assignments to eligible (just-in-time) assignments. Users activate their role when needed for a limited duration, requiring MFA and optionally approval from designated approvers. This reduces the attack surface of permanently privileged accounts and creates an audit trail of every activation.

### Q3
**Question:** A security engineer reviews Microsoft Entra ID Sign-in logs and finds multiple failed sign-in attempts for valid user accounts from IP addresses in a foreign country, followed by a successful sign-in. What Microsoft Entra ID Protection feature would have detected and responded to this automatically?

**Options:**
- A. Microsoft Entra Identity Governance — Access Reviews
- B. Microsoft Entra ID Protection — Risk-based Conditional Access (user risk and sign-in risk policies)
- C. Microsoft Entra External ID — B2B Collaboration invitations
- D. Microsoft Entra Connect Health monitoring

**Correct Answer:** B  
**Explanation:** Microsoft Entra ID Protection uses machine learning to detect risk events (impossible travel, anonymous IP, password spray, credential leak detection). Risk-based Conditional Access policies can automatically require MFA for medium-risk sign-ins and block or force password change for high-risk users. This provides automated, real-time response without manual investigation for every alert.

## Skill Area: Secure storage, databases, and networking (20–25%)

### Q4
**Question:** An organization must ensure that data stored in Azure Blob Storage is encrypted with customer-managed keys (CMK) stored in Azure Key Vault, and that the Key Vault itself is protected against accidental deletion. What steps are required?

**Options:**
- A. Enable Azure Storage Service Encryption with platform-managed keys, then rotate keys quarterly
- B. Configure Customer-Managed Key (CMK) encryption on the Storage Account with a Key Vault key, enable soft-delete and purge protection on the Key Vault, and assign a managed identity to the Storage Account with Key Vault Crypto User role
- C. Encrypt blobs manually using AES-256 before uploading and store encryption keys in a separate storage account
- D. Use Azure Disk Encryption with BitLocker keys stored in Key Vault for blob storage

**Correct Answer:** B  
**Explanation:** CMK for Azure Storage requires: (1) Azure Key Vault with soft-delete (90-day default) and purge protection enabled — purge protection prevents deletion of the key for the configured retention period; (2) A system-assigned or user-assigned managed identity on the Storage Account; (3) Key Vault access policy or RBAC role (Key Vault Crypto Service Encryption User) granted to the managed identity; (4) CMK configured on the Storage Account pointing to the key URI.

### Q5
**Question:** A data security policy requires that an Azure SQL Database must not be accessible from the public internet — all connections must come from within the Azure virtual network. What is the RECOMMENDED configuration?

**Options:**
- A. Configure the SQL Server firewall to allow only corporate IP addresses
- B. Deploy a Private Endpoint for the Azure SQL Database and disable public network access
- C. Enable the Azure SQL Server VNet service endpoint on the application subnet
- D. Enable Azure SQL Database Always Encrypted to protect data in transit

**Correct Answer:** B  
**Explanation:** Private Endpoints assign a private IP address from the VNet to the Azure SQL Server, routing all traffic over the Microsoft backbone without public internet exposure. After creating the Private Endpoint, public network access must be disabled on the SQL Server to enforce private-only connectivity. VNet service endpoints restrict access to specific subnets but still route over the public Azure backbone internally.

### Q6
**Question:** An Azure SQL Database stores sensitive PII. The security team requires that database administrators cannot see the actual values in the SSN column during query results, even in ad-hoc queries. Which Azure SQL feature addresses this?

**Options:**
- A. Transparent Data Encryption (TDE)
- B. Azure SQL Dynamic Data Masking
- C. Always Encrypted
- D. Row-Level Security

**Correct Answer:** C  
**Explanation:** Always Encrypted ensures that sensitive data is encrypted on the client side before being sent to the database and is only decrypted by authorized client applications with access to the column master key. Database admins, DBAs, and cloud operators who have access to the database server CANNOT see the plaintext values. Dynamic Data Masking only masks data in query results for unprivileged users but DBAs see full data.

## Skill Area: Secure compute (20–25%)

### Q7
**Question:** A security engineer needs to assess the attack surface of all Azure VMs in a subscription, identify missing OS patches, and receive prioritized remediation steps for critical vulnerabilities. Which Microsoft service provides this?

**Options:**
- A. Azure Update Manager — patch compliance only
- B. Microsoft Defender for Servers — integrated Defender Vulnerability Management (DVM) with risk-based prioritization
- C. Azure Security Center Secure Score recommendations
- D. Microsoft Sentinel threat hunting queries

**Correct Answer:** B  
**Explanation:** Microsoft Defender for Servers includes Microsoft Defender Vulnerability Management (DVM), which scans VM OS and installed software for CVEs, rates vulnerabilities by risk (exploitability, CVSS, exposure), and provides prioritized remediation steps. It covers Windows and Linux VMs, Azure Arc-connected servers, and provides an asset inventory view with vulnerability exposure age.

### Q8
**Question:** A company runs containerized workloads on Azure Kubernetes Service (AKS). The security team needs to ensure that container images are scanned for vulnerabilities before deployment AND that running containers are monitored for runtime threats. Which Azure service provides BOTH capabilities?

**Options:**
- A. Azure Container Registry (ACR) content trust
- B. Microsoft Defender for Containers
- C. Azure Policy for AKS (Azure Policy Add-on)
- D. Microsoft Defender for Servers with the AKS extension

**Correct Answer:** B  
**Explanation:** Microsoft Defender for Containers provides: (1) Image vulnerability assessment — scans images in ACR for CVEs at push time and on a schedule; (2) Run-time threat protection — detects threats in running AKS pods using anomaly detection and known attack pattern rules; (3) Kubernetes security posture — evaluates cluster configurations against CIS benchmarks via Azure Policy integration. This is the comprehensive Defender plan for container workloads.

## Skill Area: Manage and monitor security posture (30–35%)

### Q9
**Question:** A cloud security team wants a single score that quantifies the overall security posture of their Azure environment, shows which recommendations have the highest impact on improving the score, and tracks progress over time. Which feature provides this?

**Options:**
- A. Azure Advisor Score
- B. Microsoft Defender for Cloud Secure Score
- C. Microsoft Sentinel Incident Severity Dashboard
- D. Azure Policy Compliance Percentage

**Correct Answer:** B  
**Explanation:** Microsoft Defender for Cloud Secure Score aggregates security recommendations across compute, network, data, identity, and application resources into a single percentage score (0–100%). Each recommendation has a score impact. Remediating high-impact recommendations increases the score. The score is tracked over time and can be compared across subscriptions in a Management Group.

### Q10
**Question:** A security operations team receives 500+ alerts per day in Microsoft Sentinel. They want to automatically group related alerts into incidents and correlate them with threat intelligence to identify high-priority threats. Which Sentinel capability handles this?

**Options:**
- A. Custom KQL detection rules with email notifications
- B. Analytics rules (scheduled, ML-based, and fusion rules) with automated alert grouping into incidents
- C. Microsoft Sentinel Workbooks with alert count dashboards
- D. Data connectors with real-time alert forwarding

**Correct Answer:** B  
**Explanation:** Microsoft Sentinel Analytics rules detect threats from ingested logs and create alerts. The Fusion ML rule automatically correlates low-fidelity alerts across different data sources into high-confidence, multi-stage attack incidents (reducing alert fatigue). Threat Intelligence platform connectors enrich incidents with IOC context. Automation rules and Playbooks handle automated triage and response.

### Q11
**Question:** A security engineer must ensure that AI workloads deployed on Azure AI Foundry comply with content safety policies — specifically, that the AI model never generates harmful content, discusses restricted topics, or is manipulated via prompt injection. Which Azure service provides content safety guardrails for AI models?

**Options:**
- A. Azure OpenAI Service system prompt instructions only
- B. Azure AI Content Safety service (content filters, prompt shields, groundedness detection)
- C. Microsoft Defender for Cloud AI workload protection only
- D. Azure Policy with custom AI compliance initiative

**Correct Answer:** B  
**Explanation:** Azure AI Content Safety is a purpose-built service for AI content moderation. It provides: (1) Content filters — detect and block harmful categories (violence, hate, sexual, self-harm) in prompts and completions; (2) Prompt Shields — detect jailbreak attempts and indirect prompt injection from documents or external data; (3) Groundedness detection — identify hallucinated content not supported by retrieved documents in RAG pipelines. This is the AI-specific security domain that differentiates SC-500 from AZ-500.

### Q12
**Question:** After enabling Microsoft Defender for Cloud, a security engineer notices that several Azure SQL Databases have an alert for "Unusual export activity" and "Potential SQL injection." What is the RECOMMENDED next step using Microsoft Defender for Cloud's investigation workflow?

**Options:**
- A. Immediately disable the SQL Server to prevent further access
- B. Open the Defender for Cloud security alert, review the alert details and evidence (query text, source IP), investigate related resources using the Defender for Cloud investigation path, and trigger a Playbook for automated response if warranted
- C. Run an Azure Policy compliance scan to check SQL configurations
- D. Enable Transparent Data Encryption (TDE) on the affected databases

**Correct Answer:** B  
**Explanation:** Defender for Cloud security alerts include detailed evidence (attacker IP, query text, affected resource) and an investigation graph showing related entities. The recommended workflow is: (1) Review alert details and evidence; (2) Use the investigation path to see related entities and determine blast radius; (3) Remediate based on findings (revoke access, patch, change credentials); (4) Optionally trigger a Sentinel Playbook for automated containment. Immediately disabling the server could disrupt legitimate users.
