## Skill Area: Manage Azure identities and governance (20–25%)

### Q1
**Question:** A user needs read-only access to all resources within a specific subscription but must not be able to make any changes. Which built-in RBAC role should be assigned?

**Options:**
- A. Contributor
- B. Owner
- C. Reader
- D. User Access Administrator

**Correct Answer:** C  
**Explanation:** The Reader role grants view-only access to all resources within the assigned scope without the ability to create, modify, or delete anything. Contributor can create and manage resources but cannot grant access. Owner has full control including access management.

---

### Q2
**Question:** An organization requires that users who are members of the "Finance" group can only access Azure resources from specific IP address ranges. Which Microsoft Entra ID feature enforces this?

**Options:**
- A. Multi-Factor Authentication (MFA) only
- B. Conditional Access Policies
- C. Azure AD Application Proxy
- D. Privileged Identity Management (PIM)

**Correct Answer:** B  
**Explanation:** Conditional Access policies in Microsoft Entra ID grant or block access based on signals including user/group identity, location (IP range or named location), device compliance state, and application. A policy targeting the Finance group with a location condition restricts access to approved IP ranges.

---

### Q3
**Question:** A company uses Azure AD (Microsoft Entra ID) and wants to synchronize on-premises Active Directory users and groups to the cloud. Which service should they deploy on-premises?

**Options:**
- A. Azure AD Application Proxy
- B. Microsoft Entra Connect (Azure AD Connect)
- C. Active Directory Federation Services (AD FS)
- D. Azure Policy

**Correct Answer:** B  
**Explanation:** Microsoft Entra Connect (formerly Azure AD Connect) is an on-premises tool that synchronizes on-premises AD objects (users, groups, computers) to Microsoft Entra ID. It supports password hash synchronization, pass-through authentication, and federation with AD FS.

### Q4
**Question:** A team member leaves the organization. An administrator needs to immediately revoke all active Azure sessions for that user. What is the FASTEST way to accomplish this?

**Options:**
- A. Delete the user account from Microsoft Entra ID
- B. Remove all RBAC role assignments for the user
- C. Revoke all sign-in sessions and invalidate refresh tokens from the user's Microsoft Entra ID profile
- D. Reset the user's password and disable their account simultaneously

**Correct Answer:** C  
**Explanation:** The "Revoke sessions" action in Microsoft Entra ID (and disabling the account) immediately invalidates existing access tokens and refresh tokens, forcing re-authentication for any service. Deleting the account also works but is irreversible. Removing RBAC assignments doesn't invalidate existing tokens.

---

### Q5
**Question:** An administrator wants to apply a policy that prevents the creation of any VM with a size larger than Standard_D4s_v3 in a specific resource group. Which Azure service handles this?

**Options:**
- A. Azure RBAC
- B. Azure Management Groups
- C. Azure Policy with a "deny" effect
- D. Azure Advisor

**Correct Answer:** C  
**Explanation:** Azure Policy with a "deny" effect blocks resource creation or updates that don't comply with the policy definition. A custom or built-in policy can restrict allowed VM SKUs at any scope (management group, subscription, resource group, or resource). RBAC controls WHO can perform actions, not WHAT configurations are allowed.

## Skill Area: Implement and manage storage (15–20%)

### Q6
**Question:** A developer needs to generate a URL that grants temporary read access to a specific blob in a private Azure Storage container without making the container public. What should be used?

**Options:**
- A. Access Keys
- B. Stored Access Policy only
- C. Shared Access Signature (SAS) token
- D. Microsoft Entra ID authentication

**Correct Answer:** C  
**Explanation:** A Shared Access Signature (SAS) token grants delegated, time-limited access to storage resources without exposing the account key. A SAS can be scoped to a specific blob, container, or service, with specific permissions (read/write/delete) and an expiry time. Service SAS and User Delegation SAS are the recommended types.

---

### Q7
**Question:** An organization stores large amounts of archival data that is rarely accessed. They want to minimize storage costs and can tolerate retrieval times of up to 15 hours. Which Azure Blob Storage access tier is MOST appropriate?

**Options:**
- A. Hot
- B. Cool
- C. Cold
- D. Archive

**Correct Answer:** D  
**Explanation:** The Archive tier has the lowest storage cost but highest retrieval cost and latency (rehydration takes 1–15 hours). Hot tier is for frequently accessed data with low latency. Cool tier is for infrequently accessed data stored for at least 30 days. Cold tier (added later) suits data accessed a few times per year with 90-day minimum retention.

---

### Q8
**Question:** An administrator needs to replicate an Azure Storage account's data to a secondary region for disaster recovery, with the ability to read data from the secondary region even if the primary is available. Which replication option should be configured?

**Options:**
- A. Locally Redundant Storage (LRS)
- B. Zone-Redundant Storage (ZRS)
- C. Geo-Redundant Storage (GRS)
- D. Read-Access Geo-Redundant Storage (RA-GRS)

**Correct Answer:** D  
**Explanation:** RA-GRS replicates data asynchronously to a paired region AND provides a read endpoint for the secondary location, even when the primary is healthy. GRS replicates to the secondary but only allows reads during a failover. LRS stays within one datacenter. ZRS replicates across Availability Zones within a single region.

---

### Q9
**Question:** A company needs to lift-and-shift an on-premises Windows file server to Azure so that existing applications continue using the SMB protocol without modification. Which Azure service best meets this need?

**Options:**
- A. Azure Blob Storage
- B. Azure Files
- C. Azure NetApp Files
- D. Azure Queue Storage

**Correct Answer:** B  
**Explanation:** Azure Files provides fully managed cloud file shares accessible via SMB 3.x and NFS 4.1 protocols. Existing applications can mount Azure file shares using the same SMB paths they used on-premises. Azure Blob Storage uses a proprietary REST API, not SMB.

## Skill Area: Deploy and manage Azure compute resources (20–25%)

### Q10
**Question:** An administrator deploys a Virtual Machine Scale Set (VMSS) to handle variable web traffic. The team wants VMs to be added when average CPU exceeds 75% for 5 minutes and removed when CPU drops below 30% for 10 minutes. What feature of VMSS enables this?

**Options:**
- A. Availability Sets
- B. Autoscale rules
- C. Azure Load Balancer health probes
- D. Azure Policy remediation tasks

**Correct Answer:** B  
**Explanation:** VMSS Autoscale rules define scale-out and scale-in conditions based on metrics (CPU, memory, custom metrics) with configurable thresholds and time windows. The VMSS automatically adjusts instance count. Health probes determine which instances receive traffic but don't control scaling.

---

### Q11
**Question:** A team needs to deploy a containerized application to Azure without managing Kubernetes clusters. The application consists of a single container that runs continuously. Which service is MOST appropriate?

**Options:**
- A. Azure Kubernetes Service (AKS)
- B. Azure Container Instances (ACI)
- C. Azure Container Apps
- D. Azure App Service with Docker

**Correct Answer:** B  
**Explanation:** Azure Container Instances (ACI) is the fastest and simplest way to run a single container in Azure without managing any orchestration infrastructure. It's ideal for simple, isolated workloads. Azure Container Apps adds serverless scaling and is better for microservices. AKS is for complex multi-service applications requiring full Kubernetes capabilities.

---

### Q12
**Question:** An administrator needs to run a script that installs software on an Azure VM immediately after it is deployed, without using a configuration management tool. Which VM feature handles this?

**Options:**
- A. Azure VM Extension (Custom Script Extension)
- B. Azure Automation State Configuration
- C. Azure Monitor Agent
- D. Azure Bastion

**Correct Answer:** A  
**Explanation:** The Custom Script Extension downloads and executes scripts on Azure VMs during or after deployment. It runs as part of VM provisioning and can install software, configure OS settings, and perform post-deployment tasks. Azure Automation State Configuration (DSC) is for ongoing compliance, not one-time scripts.

## Skill Area: Implement and manage virtual networking (15–20%)

### Q13
**Question:** Two Azure Virtual Networks in different regions need to communicate with each other using private IP addresses without routing traffic through the public internet. Which Azure networking feature provides this?

**Options:**
- A. Virtual Network Peering (Global VNet Peering)
- B. Azure VPN Gateway
- C. Azure ExpressRoute
- D. Service Endpoints

**Correct Answer:** A  
**Explanation:** Global VNet Peering connects Azure Virtual Networks across regions using the Microsoft backbone network with low latency and no public internet exposure. Traffic travels over private IP addresses. VPN Gateway is an option but routes traffic through gateway VMs, adding latency and cost.

---

### Q14
**Question:** A company wants to prevent all internet traffic from reaching a subnet that hosts a database server, while still allowing traffic from the application tier subnet. What is the MOST appropriate solution?

**Options:**
- A. Route all database subnet traffic through an Azure Firewall
- B. Configure a Network Security Group (NSG) on the database subnet with a deny rule for internet inbound traffic
- C. Enable Azure DDoS Protection Standard on the virtual network
- D. Configure Service Endpoints on the database subnet

**Correct Answer:** B  
**Explanation:** Network Security Groups (NSGs) contain security rules that allow or deny inbound/outbound network traffic to Azure resources. Applying an NSG to the database subnet with a rule denying inbound traffic from the internet (source: Any or Internet) while allowing traffic from the app subnet is the standard and most cost-effective approach.

---

### Q15
**Question:** An administrator needs to resolve DNS names for Azure resources within a virtual network using custom domain names (e.g., contoso.internal) without deploying their own DNS servers. Which Azure service supports this?

**Options:**
- A. Azure DNS Public Zones
- B. Azure Private DNS Zones
- C. Azure Traffic Manager
- D. Azure VPN Gateway DNS forwarding

**Correct Answer:** B  
**Explanation:** Azure Private DNS Zones provide DNS resolution for virtual networks without requiring custom DNS server VMs. A private zone (e.g., contoso.internal) is linked to the VNet, and Azure auto-registers VM hostnames or you create manual A records. Public DNS Zones are for internet-facing name resolution.

## Skill Area: Monitor and maintain Azure resources (10–15%)

### Q16
**Question:** An administrator wants to receive an email alert when the CPU percentage on a production VM exceeds 90% for more than 5 minutes. Which Azure Monitor components are needed?

**Options:**
- A. A Log Analytics workspace and a KQL query alert
- B. An Azure Monitor Metric Alert rule with an Action Group configured with an email action
- C. Microsoft Defender for Cloud recommendations
- D. Azure Advisor cost recommendations

**Correct Answer:** B  
**Explanation:** Azure Monitor Metric Alerts evaluate platform metrics (like CPU percentage) in near real-time at defined intervals. When the metric breaches the threshold for the configured duration, the alert fires an Action Group that can send emails, SMS, call webhooks, trigger Logic Apps, and more.

---

### Q17
**Question:** A production Azure SQL Database was accidentally deleted. The database had automated backups enabled with a 35-day retention period. How can the administrator restore the database?

**Options:**
- A. Restore from a manual backup snapshot taken before deletion
- B. Use Azure Site Recovery to recover the database
- C. Use the Point-in-Time Restore (PITR) feature to restore to any point within the retention period
- D. Contact Microsoft Support to recover the deleted database

**Correct Answer:** C  
**Explanation:** Azure SQL Database automated backups support Point-in-Time Restore (PITR), allowing administrators to restore to any second within the backup retention period (7–35 days). Deleted databases can be restored within the retention window. PITR creates a new database; it doesn't overwrite the original.

---

### Q18
**Question:** An administrator needs to query logs from multiple Azure resources (VMs, storage accounts, networking) in a single interface using a structured query language. Which Azure service provides this?

**Options:**
- A. Azure Monitor Metrics Explorer
- B. Azure Log Analytics (with KQL)
- C. Microsoft Defender for Cloud
- D. Azure Service Health

**Correct Answer:** B  
**Explanation:** Azure Log Analytics is a workspace within Azure Monitor that collects and stores log data from multiple sources. Using Kusto Query Language (KQL), administrators can write queries to filter, aggregate, and analyze logs across all connected resources. Metrics Explorer handles numeric time-series data, not log records.
