## Skill Area: Design identity, governance, and monitoring solutions (25–30%)

### Q1
**Question:** An enterprise has 12 Azure subscriptions organized into three business units. The security team needs to apply a consistent set of compliance policies across ALL subscriptions and receive a unified compliance dashboard. What is the BEST architectural approach?

**Options:**
- A. Apply Azure Policy to each subscription individually
- B. Create a Management Group hierarchy and apply Azure Policy at the root Management Group level
- C. Create a separate Azure Policy initiative for each business unit
- D. Use Microsoft Defender for Cloud recommendations per subscription

**Correct Answer:** B  
**Explanation:** Management Groups provide a hierarchy above subscriptions. Azure Policy and Azure Blueprints applied at a Management Group scope are inherited by all child subscriptions. This ensures consistent governance without duplicating policy assignments. The Microsoft Defender for Cloud regulatory compliance dashboard can also aggregate across subscriptions when linked to the same Management Group.

### Q2
**Question:** A solutions architect needs to design access management for an application that must read secrets from Azure Key Vault without storing credentials in code or configuration files. Which identity approach is recommended by the Well-Architected Framework?

**Options:**
- A. Store the Key Vault access key in the application's environment variables
- B. Create a service principal with a client secret and rotate it every 30 days
- C. Use a Managed Identity assigned to the compute resource (VM, App Service, or AKS pod) and grant it Key Vault access via RBAC
- D. Use the Key Vault reference feature with an App Service connection string

**Correct Answer:** C  
**Explanation:** Managed Identities eliminate credential management entirely. The identity is managed by Azure AD; the compute resource obtains access tokens automatically without storing secrets. This is the Zero Trust–aligned best practice. Client secrets require rotation management and introduce credential leak risk.

### Q3
**Question:** A company must demonstrate compliance with ISO 27001, SOC 2, and PCI DSS across their Azure environment. The compliance team wants a centralized view of compliance status per regulatory framework. Which Azure capability provides this?

**Options:**
- A. Azure Advisor security score
- B. Microsoft Defender for Cloud — Regulatory Compliance dashboard
- C. Azure Policy compliance overview
- D. Azure Blueprints with compliance assignment

**Correct Answer:** B  
**Explanation:** Microsoft Defender for Cloud's Regulatory Compliance dashboard maps Azure resource configurations against specific regulatory frameworks (ISO 27001, SOC 2, PCI DSS, NIST, CIS, etc.) and shows passing/failing controls in a unified view. Azure Policy shows compliance against individual policy definitions, not regulatory frameworks.

### Q4
**Question:** An architect is designing the monitoring strategy for a multi-region application. They need to correlate telemetry (requests, dependencies, exceptions, performance counters) from App Service, Azure Functions, and AKS into a single observability platform. Which service should be the central collection point?

**Options:**
- A. Azure Monitor Metrics only
- B. Application Insights connected to a Log Analytics workspace
- C. Azure Network Watcher
- D. Azure Service Health

**Correct Answer:** B  
**Explanation:** Application Insights is the application performance monitoring (APM) layer of Azure Monitor. It collects distributed traces, dependency maps, exceptions, and custom telemetry from any application. When backed by a Log Analytics workspace, all telemetry is queryable with KQL alongside infrastructure logs, enabling end-to-end correlation across services and regions.

## Skill Area: Design data storage solutions (25–30%)

### Q5
**Question:** An e-commerce application needs a database that can handle 50,000 transactions per second globally, with data replicated to five regions with active-active write capability and a maximum tolerable data loss of zero seconds. Which Azure database service meets these requirements?

**Options:**
- A. Azure SQL Database Hyperscale
- B. Azure Cosmos DB with multi-region writes enabled
- C. Azure Database for PostgreSQL — Flexible Server with read replicas
- D. Azure SQL Managed Instance Business Critical

**Correct Answer:** B  
**Explanation:** Azure Cosmos DB is the only Azure database with native multi-region, multi-master (active-active) write capability with configurable consistency levels. With Strong consistency, RPO approaches zero. It scales throughput independently with Request Units (RUs) to handle any transaction volume. SQL Hyperscale supports one writable primary; PostgreSQL read replicas are read-only.

### Q6
**Question:** A data analytics team needs to store petabytes of structured and semi-structured data in a single storage layer that supports both SQL queries and Apache Spark processing. Which storage architecture should an architect recommend?

**Options:**
- A. Azure Blob Storage (hot tier) with Azure Data Factory pipelines
- B. Azure Data Lake Storage Gen2 as the foundation for a lakehouse architecture on Azure Synapse Analytics or Microsoft Fabric
- C. Azure SQL Database Hyperscale with external tables
- D. Azure Cosmos DB with analytical store enabled

**Correct Answer:** B  
**Explanation:** Azure Data Lake Storage Gen2 (hierarchical namespace on blob storage) provides the scalable, cost-effective foundation for a lakehouse. Azure Synapse Analytics (SQL Pools + Spark Pools) or Microsoft Fabric can query and process data directly from ADLS Gen2. This is the standard enterprise analytics architecture for petabyte-scale workloads.

### Q7
**Question:** A financial services company needs to move 500 TB of on-premises data to Azure Blob Storage. Their internet connection is 1 Gbps, and they need the transfer to complete within 2 weeks. Which data transfer service should an architect recommend?

**Options:**
- A. AzCopy over the internet with parallel uploads
- B. Azure Data Factory with self-hosted integration runtime
- C. Azure Data Box (80 TB appliance) or Azure Data Box Disk
- D. Azure ExpressRoute (10 Gbps circuit) provisioned for the transfer period

**Correct Answer:** C  
**Explanation:** 500 TB over a 1 Gbps connection would take approximately 44 days (500TB / 125 MB/s). Azure Data Box ships a physical appliance (80 TB or 100 TB usable) to the customer's site. Data is copied offline and shipped back to Microsoft for upload. For 500 TB, 6-7 Data Box units or the 1 PB Data Box Heavy would be used. This is always the recommendation when network transfer time exceeds ~2 weeks.

### Q8
**Question:** An architect needs to design a data integration solution that ingests data from 20 different source systems (on-premises SQL, REST APIs, Salesforce) and loads it into Azure Data Lake Storage Gen2 on a scheduled basis. Which Azure service is purpose-built for this?

**Options:**
- A. Azure Logic Apps
- B. Azure Event Hubs
- C. Azure Data Factory
- D. Azure Stream Analytics

**Correct Answer:** C  
**Explanation:** Azure Data Factory (ADF) is the cloud-native ETL/ELT and data integration service with 90+ connectors for databases, SaaS apps, REST APIs, and file systems. It supports scheduled, trigger-based, and event-driven pipelines. Azure Logic Apps is for workflow automation, not large-scale data movement. Event Hubs and Stream Analytics are for real-time streaming, not batch ETL.

## Skill Area: Design business continuity solutions (10–15%)

### Q9
**Question:** An organization's RTO (Recovery Time Objective) is 1 hour and RPO (Recovery Point Objective) is 15 minutes for a critical Azure VM workload. Which Azure service and configuration achieves this with the lowest cost?

**Options:**
- A. Azure Backup with daily backups and Azure Site Recovery (ASR) with 5-minute replication
- B. Azure Site Recovery alone with 5-minute RPO replication to a secondary region
- C. Geo-Redundant Storage for VM disks with manual failover scripts
- D. Azure Backup with 4-hour backup frequency

**Correct Answer:** B  
**Explanation:** Azure Site Recovery (ASR) continuously replicates VMs to a secondary region and achieves RPO as low as 30 seconds for VMware/physical and ~5 minutes for Azure VMs. RTO depends on failover execution time (typically 30–60 minutes). ASR alone meets the stated RTO/RPO without requiring backup. Azure Backup is for point-in-time recovery, not disaster recovery failover.

### Q10
**Question:** When designing a high-availability architecture for an Azure SQL Database that requires 99.99% SLA, read scale-out for reporting queries, and automatic failover across Availability Zones, which service tier should the architect select?

**Options:**
- A. General Purpose with Geo-Redundant Backup
- B. Business Critical tier (supports Availability Zone redundancy and built-in read replicas)
- C. Hyperscale with read replicas
- D. Standard tier with Active Geo-Replication to a secondary region

**Correct Answer:** B  
**Explanation:** Azure SQL Database Business Critical tier includes a built-in Always On availability group with 3–4 replicas across Availability Zones, provides 99.99% SLA, and exposes one readable secondary replica for scale-out. It's designed for the highest HA requirements. Hyperscale supports read replicas but has different HA architecture and SLA characteristics.

### Q11
**Question:** An organization needs a backup solution for Azure Files shares that meets compliance requirements of 7-year retention with immutable backup copies. What should the architect design?

**Options:**
- A. Azure Backup for Azure Files with soft-delete enabled and long-term retention policies in a Recovery Services Vault
- B. Manual AzCopy scripts to copy file share snapshots to a separate storage account weekly
- C. Azure File Sync with cloud tiering to an on-premises server as backup
- D. Azure Data Factory to export file share contents to Azure Blob Storage (Archive tier) nightly

**Correct Answer:** A  
**Explanation:** Azure Backup for Azure Files integrates with Recovery Services Vault to provide scheduled snapshots, long-term retention policies (daily/weekly/monthly/yearly up to 99 years), and soft-delete protection against accidental deletion. Vault immutability settings make backup data tamper-resistant for compliance. Manual scripts lack the auditability and policy enforcement required for compliance.

## Skill Area: Design infrastructure solutions (25–30%)

### Q12
**Question:** An architect is designing a microservices application that requires automatic scaling based on HTTP request rate, built-in service discovery, Dapr integration, and no Kubernetes cluster management overhead. Which Azure service best fits?

**Options:**
- A. Azure Kubernetes Service (AKS)
- B. Azure Container Apps
- C. Azure App Service Plan (Premium)
- D. Azure Spring Apps

**Correct Answer:** B  
**Explanation:** Azure Container Apps is built on Kubernetes and KEDA (Kubernetes Event-Driven Autoscaling) but abstracts away cluster management. It supports HTTP-based autoscaling from 0 to N, built-in service discovery via environment DNS, native Dapr integration, and revision management. AKS gives full Kubernetes control but requires cluster administration overhead.

### Q13
**Question:** A company is migrating a legacy on-premises application to Azure. The application requires Windows Server 2012 R2 (end of support), has an SQL Server 2014 database, and the team wants to minimize code changes. Which migration strategy aligns with the Azure Well-Architected Framework recommendation for minimizing risk?

**Options:**
- A. Rewrite the application as a cloud-native microservices architecture
- B. Lift-and-shift the VMs to Azure IaaS and enroll them in Azure Extended Security Updates (ESU)
- C. Re-platform to Azure App Service and Azure SQL Database
- D. Containerize the application immediately and deploy to AKS

**Correct Answer:** B  
**Explanation:** Lift-and-shift (Rehost) to Azure VMs minimizes risk and change effort for legacy applications. Azure provides free Extended Security Updates (ESU) for Windows Server 2012 R2 and SQL Server 2014 hosted on Azure, extending security patch coverage. This is the lowest-risk migration path when code changes are not feasible.

### Q14
**Question:** An architect must design a network topology for an enterprise with 50 Azure virtual networks across 8 regions, centralized security inspection for all east-west and north-south traffic, and spoke VNets that cannot communicate directly with each other. Which topology meets these requirements?

**Options:**
- A. Full mesh VNet peering with NSGs on every subnet
- B. Hub-and-spoke topology with Azure Firewall deployed in the hub and UDRs forcing all spoke traffic through the hub
- C. Azure Virtual WAN with Secured Hub (Azure Firewall integrated)
- D. Both B and C are valid architectural options — Azure Virtual WAN is preferred at this scale

**Correct Answer:** D  
**Explanation:** Both hub-and-spoke with Azure Firewall and Azure Virtual WAN with Secured Hubs implement centralized traffic inspection and prevent direct spoke-to-spoke communication. At 50 VNets across 8 regions, Azure Virtual WAN is architecturally preferred because it manages inter-hub routing automatically, whereas traditional hub-and-spoke requires manual route management across regions. The exam accepts both as valid designs; the key concepts are hub firewall placement and UDR/route table configuration.
