## Skill Area: Describe AI concepts and Azure AI services (25–30%)

### Q1
**Question:** Which of the following best describes the difference between AI and traditional software?

**Options:**
- A. AI software uses rule-based logic explicitly programmed by developers; traditional software learns from data
- B. AI systems learn patterns from data to make predictions or decisions; traditional software executes explicit rules programmed by developers
- C. AI is only applicable to image recognition; traditional software handles text and logic
- D. AI requires no data — it is based purely on mathematical formulas

**Correct Answer:** B  
**Explanation:** Traditional software executes deterministic rules written by programmers. AI (specifically machine learning) learns patterns from data to generalize to new inputs. A spam filter built with traditional software has explicit keyword rules; an ML-based spam filter learns which patterns correlate with spam from historical email data.

### Q2
**Question:** What is Azure AI Foundry?

**Options:**
- A. A standalone programming language for building AI models
- B. Microsoft's unified platform for building, deploying, and managing enterprise-grade AI applications and agents, including model catalog, project management, evaluation, and deployment infrastructure
- C. A database service for storing AI model weights
- D. A code editor with AI autocomplete features

**Correct Answer:** B  
**Explanation:** Azure AI Foundry (formerly Azure AI Studio) is the end-to-end platform for enterprise AI development. It provides: the Model Catalog (hundreds of foundation models from OpenAI, Meta, Mistral, etc.), project management with Hub-Project hierarchy, prompt flow for LLMOps, evaluation tools, deployment to managed compute or serverless endpoints, and integration with Azure AI services.

## Skill Area: Describe generative AI capabilities on Azure (30–35%)

### Q4
**Question:** What is "prompt engineering" in the context of working with large language models?

**Options:**
- A. Designing the hardware infrastructure that runs LLM inference
- B. The practice of crafting and refining the text input (prompt) to an LLM to elicit the desired output quality, format, and behavior
- C. Training a new language model from scratch
- D. Encoding data as numerical vectors for model input

**Correct Answer:** B  
**Explanation:** Prompt engineering is the discipline of designing effective prompts for LLMs. Techniques include zero-shot (instructions only), few-shot (examples in the prompt), chain-of-thought (ask the model to reason step by step), system prompts (behavioral instructions), and structured output formatting. The quality of the prompt strongly determines the quality of the model's output.

### Q5
**Question:** A company wants to use an LLM to answer questions about their product catalog (10,000 items) which is updated daily. Which approach best keeps answers accurate and current?

**Options:**
- A. Fine-tune GPT-4o on the product catalog data monthly
- B. RAG (Retrieval-Augmented Generation) — index the catalog in Azure AI Search, retrieve relevant items at query time, and include them in the prompt
- C. Include the entire 10,000-item catalog in every system prompt
- D. Train a custom small language model specifically on product data

**Correct Answer:** B  
**Explanation:** RAG retrieves only the relevant subset of documents at query time, keeping answers current (re-index when catalog updates) without retraining. Fine-tuning is expensive, slow (hours to days), and doesn't keep pace with daily updates — it teaches style/format, not updatable factual knowledge. Including the full catalog in every prompt exceeds context window limits and is prohibitively expensive.

### Q6
**Question:** What are "tokens" in the context of LLMs, and why do they matter for Azure OpenAI Service pricing?

**Options:**
- A. Authentication credentials required to call the OpenAI API
- B. The basic units of text that LLMs process (roughly 4 characters or 0.75 words in English); Azure OpenAI pricing is based on the number of input and output tokens consumed
- C. Individual words that the model processes — one word equals one token
- D. API calls to Azure OpenAI Service — pricing is per call

**Correct Answer:** B  
**Explanation:** Tokens are the subword units that LLMs process. Tokenization splits text into tokens (a word may be 1–3 tokens; common words are typically 1 token, rare or complex words may be 2–3). Azure OpenAI prices per 1,000 or 1,000,000 tokens for input (prompt) and output (completion) separately. Longer prompts (from RAG context or few-shot examples) consume more tokens and increase cost.

### Q7
**Question:** Which Azure OpenAI model is best suited for generating images from text descriptions?

**Options:**
- A. GPT-4o
- B. Whisper
- C. DALL-E 3
- D. text-embedding-ada-002

**Correct Answer:** C  
**Explanation:** DALL-E 3 is Azure OpenAI's image generation model that creates images from natural language descriptions (text-to-image). GPT-4o is a multimodal text and image understanding model (but generates text, not images, by default). Whisper is a speech-to-text transcription model. text-embedding-ada-002 is an embedding model that converts text to numeric vectors.

## Skill Area: Describe responsible AI principles and practices (20–25%)

### Q8
**Question:** A hiring algorithm is found to recommend fewer female candidates for senior engineering roles than equally qualified male candidates. Which Responsible AI principle is being violated?

**Options:**
- A. Reliability
- B. Fairness
- C. Transparency
- D. Privacy

**Correct Answer:** B  
**Explanation:** Fairness requires that AI systems don't discriminate against people based on demographic characteristics (gender, race, age, disability). A hiring algorithm exhibiting gender bias violates Fairness. Microsoft's Responsible AI framework includes tooling (Fairlearn, Responsible AI Dashboard in Azure ML) to detect and mitigate bias in ML models.

### Q9
**Question:** An AI engineer wants to understand WHY a machine learning model predicted "high risk" for a specific loan application. Which practice aligns with the Transparency and Accountability Responsible AI principles?

**Options:**
- A. Deploying a black-box neural network without explanation features
- B. Using Explainable AI (XAI) techniques such as SHAP values or LIME to show which features most influenced the prediction
- C. Averaging predictions across 10 models (ensemble) to reduce individual model bias
- D. Encrypting the model to protect intellectual property

**Correct Answer:** B  
**Explanation:** Explainable AI (XAI) techniques like SHAP (SHapley Additive exPlanations) calculate each feature's contribution to a specific prediction. This enables transparency ("the loan was flagged as high risk because monthly debt-to-income ratio was the top factor") and accountability (decisions can be audited and challenged). Azure Machine Learning's Responsible AI Dashboard integrates SHAP explanations.

## Skill Area: Describe agentic AI concepts (15–20%)

### Q10
**Question:** What is an "AI agent" in the context of Azure AI Foundry?

**Options:**
- A. A human employee who reviews and approves AI-generated content
- B. An AI system that can perceive its environment, reason about goals, use tools, and take autonomous multi-step actions to accomplish tasks
- C. A software bot that can only answer predefined FAQ questions
- D. A real-time data streaming pipeline

**Correct Answer:** B  
**Explanation:** AI agents (in the AI Foundry sense) go beyond simple question-answering. They perceive inputs (user messages, tool results, documents), reason using an LLM to plan actions, use tools (search, code execution, API calls, file operations) to gather information or take actions, and iterate until the goal is accomplished. This agentic loop (perceive → reason → act → observe) enables autonomous multi-step task completion.

### Q11
**Question:** Which of the following is an example of an AI agent using a "tool"?

**Options:**
- A. An LLM generating a text completion based on its training knowledge
- B. An AI agent calling an API to retrieve the current stock price before including it in a financial analysis response
- C. A machine learning model making a classification prediction
- D. A speech model converting audio to text

**Correct Answer:** B  
**Explanation:** Tools extend an AI agent's capabilities beyond its training knowledge. When an agent calls an API (stock price), runs code, searches the web, queries a database, or writes a file, it is using a tool. Azure AI Foundry Agents support built-in tools (code interpreter, file search, Bing grounding) and custom function tools defined by the developer.

### Q12
**Question:** What is "multi-agent orchestration" in the context of AI systems?

**Options:**
- A. Running multiple instances of the same AI model for load balancing
- B. Coordinating multiple specialized AI agents, each with distinct skills or knowledge, to collaborate on complex tasks that exceed any single agent's capabilities
- C. Training multiple models and selecting the best performing one
- D. Using multiple prompts to fine-tune a single language model

**Correct Answer:** B  
**Explanation:** Multi-agent architectures assign specialized roles to individual agents (e.g., a research agent, a coding agent, a review agent) and coordinate them via an orchestrator. The orchestrator decomposes tasks, routes subtasks to the appropriate specialist agent, and aggregates results. Frameworks like Semantic Kernel, AutoGen, and Azure AI Foundry support multi-agent patterns.
